# Environment Variables Reference

All env vars are read in `api/src/main.rs` at startup.

## Quick Start

```bash
cd api
cp config/.env.example config/.env
# Edit config/.env with your values
cargo run
```

## Variables

| Variable | Required | Default | Used When | Description |
|---|---|---|---|---|
| `PORT` | No | `3001` | Always | HTTP port |
| `RUST_LOG` | No | _(none)_ | Always | Log level (`error`, `warn`, `info`, `debug`, `trace`) |
| `DATA_SOURCE` | No | `local` | Always | `local` or `s3` |
| `DATA_PATH` | No | `output` | `DATA_SOURCE=local` | Path to solver output directory |
| `AWS_REGION` | No | `ap-southeast-1` | `DATA_SOURCE=s3` | AWS region |
| `AWS_ACCESS_KEY_ID` | No | _(auto)_ | `DATA_SOURCE=s3` | IAM access key (or use instance role) |
| `AWS_SECRET_ACCESS_KEY` | No | _(auto)_ | `DATA_SOURCE=s3` | IAM secret key (or use instance role) |
| `S3_BUCKET` | **Yes** | - | `DATA_SOURCE=s3` | S3 bucket name (panics if missing) |
| `S3_PREFIX` | No | `output` | `DATA_SOURCE=s3` | Key prefix in bucket |

## Where to Get Each Value

### PORT
Any available port on your machine. Default `3001` matches the GTO Wizard preflop API convention.

### RUST_LOG
Set to `info` for normal operation. Use `debug` for request-level logging.

### DATA_SOURCE
- `local` for development (reads files from disk)
- `s3` for production (reads from AWS S3)

### DATA_PATH
Path to the directory containing solver output folders. Each folder is named by stacks divided by 10M, space-separated (e.g., `10 17 54 17 15 32 21 29`).

```
{DATA_PATH}/
  {stack_folder}/
    settings.json    # Solver config: stacks, blinds, equity model
    equity.json      # ICM equity: pre-hand equity, bubble factors
    nodes/
      0.json         # Root node (UTG to act)
      1.json         # After UTG folds
      ...            # ~3000 nodes per configuration
```

**Source:** Generated by the GTO solver. See `output/` in the spec project for an example.

### AWS_REGION
Where your S3 bucket is hosted. Find it in:
- AWS Console -> S3 -> select bucket -> Properties -> AWS Region

### AWS_ACCESS_KEY_ID / AWS_SECRET_ACCESS_KEY
IAM credentials with `s3:GetObject` and `s3:ListBucket` permissions.

**Where to create:**
- AWS Console -> IAM -> Users -> your user -> Security credentials -> Create access key

**On AWS infrastructure** (EC2, ECS, Lambda): leave blank. The AWS SDK automatically uses the instance/task role.

### S3_BUCKET
The bucket name (not ARN, not URL). Find it in:
- AWS Console -> S3 -> bucket list

### S3_PREFIX
The key prefix before the stack folders in S3. If your data is at:
```
s3://my-bucket/output/10 17 54 17 15 32 21 29/nodes/0.json
```
Then `S3_BUCKET=my-bucket` and `S3_PREFIX=output`.

## Load Order

The server reads env vars in priority order (last wins):

1. `config/.env` file
2. `.env` file at project root
3. System environment variables

Code: `main.rs` lines 24-25 (`dotenvy::from_filename` then `dotenvy::dotenv`)

## Example Configurations

### Local development
```env
PORT=3001
RUST_LOG=info
DATA_SOURCE=local
DATA_PATH=../output
```

### S3 production (with IAM keys)
```env
PORT=8080
RUST_LOG=warn
DATA_SOURCE=s3
AWS_REGION=ap-southeast-1
AWS_ACCESS_KEY_ID=AKIA...
AWS_SECRET_ACCESS_KEY=wJal...
S3_BUCKET=my-gto-solver-data
S3_PREFIX=output
```

### S3 production (with instance role on EC2/ECS)
```env
PORT=8080
RUST_LOG=warn
DATA_SOURCE=s3
AWS_REGION=ap-southeast-1
S3_BUCKET=my-gto-solver-data
S3_PREFIX=output
```
